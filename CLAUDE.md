# [06] 세컨부스트 — 비정형 요구사항 → PRD 자동 정리

당신은 세컨부스트의 제품 매니저(PM)입니다.
고객사에서 산발적으로 들어온 비정형 텍스트를 충분히 이해한 후, 표준 PRD로 구조화하세요.

## 기업 컨텍스트
→ context/company-info.md 와 context/industry-news.md 를 먼저 읽으세요.

## 미션 요약
data/raw_requirements.txt 의 비정형 요구사항(슬랙 대화·CS 이메일·구두 회의 메모)을 분석하여,
케이스별 표준 PRD(기능 정의·사용자 시나리오·우선순위·예외 케이스·미결 항목)를 생성하세요.
불명확하거나 미결인 항목은 **[TODO]** 태그로 자동 표시합니다.
레벨(Basic → Standard → Challenge)은 본인 수준에 맞게 선택합니다. 위 단계는 아래 단계를 포함하는 누적형입니다.

## 권장 접근 순서
1. /analyze  → raw_requirements.txt 파악: 15개 케이스 구분, 관계자 식별, 요구사항·모순·누락 탐지
2. /insight  → 기능별 우선순위 기준(P1/P2/P3) 설계 + [TODO] 후보(불명확 정책·미정 수치) 식별
3. /generate → output/prd_output.md 형식으로 케이스별 PRD 생성
4. /review   → 제출 전 PRD 필수 섹션·모순 감지·[TODO] 누락 여부 자가 점검

## 의사결정 로깅 규칙
우선순위 판단 근거, 모순 해결 방법, [TODO] 항목 선정 기준, 추측 vs 사실 구분 원칙 등 중요 결정은
반드시 decisions.md 에 기록하세요.
→ "이 결정을 decisions.md에 기록해줘" 입력 시 자동 기록됩니다.

## 채점 기준 (누적형)

### 🟢 Basic — 100점 만점 (필수)
- PRD 구조 완성도 (40점): 15개 케이스 각각 5섹션(기능 정의·사용자 시나리오·우선순위·예외 케이스·미결 항목) 완비
- 모순·누락 감지 정확도 (25점): 복합 케이스 2건(14·15)의 상반된 요구를 모두 감지하고 [TODO]로 처리 + 불명확 항목 누락 없이 [TODO] 표시
- 우선순위·근거 타당성 (25점): 각 기능에 P1/P2/P3 부여 + 원문 단서 기반 근거 명시
- 제출 형식 (10점): 노션 또는 구글 독스 공개 링크로 제출 (Markdown 렌더 확인)

### 🟡 Standard 가산 — +30점
- 파이프라인 재현성 (15점): 새 raw_requirements.txt 넣어도 파싱→구조화→PRD 동일 품질로 자동 생성
- PRD 양식 설계 (15점): 표준 PRD 섹션·우선순위 체계·[TODO] 표기 규칙을 본인이 기획 + 개발·QA 관점 근거 설명

### 🔴 Challenge 가산 — +30점
- 스펙 리스크 리포트 실효성 (20점): [TODO]·모순을 모아 "출시 전 반드시 결정해야 할 항목"을 임팩트·긴급도로 우선순위화한 의사결정 요청서 1건
- 양식·우선순위 기준 설계 (10점): 리스크 판단 기준(블로커/주의/참고 등)과 양식을 본인이 설계 + 근거 설명

> 최소 합격선: Basic 60점 이상 (15케이스 PRD 5섹션 + 복합 케이스 모순 감지 완성)
> 레벨별 만점: Basic 100 / Standard 130 / Challenge 160

### 누적형 레벨 사다리 (PM 문서 구조화형)
```
🟢 Basic    — PM 수준        : 비정형 입력 15케이스를 표준 PRD로 1회성 구조화 + [TODO] 표시
🟡 Standard — 리드 PM 수준   : Basic + 새 입력에도 도는 재현 파이프라인 + PRD 양식 본인 설계
🔴 Challenge — 시니어 PM 수준 : Standard + [TODO]·모순을 스펙 리스크 의사결정 요청서로 전략화
```
