# 문제정의 워크시트

오늘은 만든 결과물을 새로 고치지 않아요. 이미 만든 그 결과물이 **어떤 문제를 왜 풀었는지**를 더 또렷하게 다시 쓰고, 그 문장을 그대로 포트폴리오에 옮겨 담는 게 오늘 할 일이에요.

기업은 포폴을 볼 때 코드보다 **"무슨 문제를, 왜 풀었나"** 를 먼저 봐요. 그리고 이건 실제 심사에서도 그대로 보는 부분이고요(아래 표).

진행은 이렇게 해요. **① 예시를 보면서 혼자 최대한 채워보기 → ② 막히는 칸엔 ❓ 표시 → ③ 퍼실(지희·진·혜인)이 돌면서 그 지점을 "왜?"로 같이 뾰족하게** 만들어 드려요. 혼자 다 못 채워도 괜찮아요. 막힌 곳을 남겨두는 게 오늘은 잘하는 거예요.

---

## 이 워크시트가 실제 심사와 어떻게 연결되나요?

채우다 보면 자연스럽게 심사 4축이 다 채워지도록 짜여 있어요.

| 심사에서 보는 것 | 이 워크시트에서 채우는 곳     |
| ---------------- | ----------------------------- |
| ① 문제 정의     | STEP 1·2·3                  |
| ② 기술·완성도  | STEP 4 — 검증 줄             |
| ③ 주도성        | STEP 4 — 주도성 줄           |
| ④ 전달력        | STEP 5 (한 문장으로 또렷하게) |

---

## 먼저 볼까요 — 얕은 문제 vs 뾰족한 문제

같은 결과물이라도 문제를 어떻게 쓰느냐에 따라 완전히 다르게 읽혀요.

|      | 얕은 문장                                          | 뾰족한 문장 (우수 사례)                                                                                                                                                                                      |
| ---- | -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 문장 | "박람회 후 후속 이메일을 자동으로 많이 만들어준다" | "박람회 후**48시간 골든타임** 안에, **직책별 맥락과 상담 내용**을 살린 후속 메일을 보내 **리드 전환율**을 높인다. 담당자는 그 시간에 **어느 리드에 집중할지 판단**할 여유를 얻는다." |
| 차이 | 기능 설명에서 멈춤                                 | 업계 타이밍(48h) + 비즈니스 숫자(전환율) + 사람을 더 중요한 일로 재배치                                                                                                                                      |

왼쪽 문장을 오른쪽으로 옮기는 게 목표예요. 방법은 하나, **"왜?"를 세 번 묻는 것**이에요.

---

## STEP 0 — 내가 만든 것 (한 줄)

코드를 바꾸는 게 아니라, 이미 만든 걸 한 줄로 적고 시작하면 돼요.

```
내가 만든 것: _______________________________________________
```

예시 (우수 사례): "박람회 상담 메모를 넣으면 직책별로 후속 이메일 초안을 만들어주는 파이프라인"

---

## STEP 1 — "왜?" 세 번 타고 올라가기

혼자 최대한 채워보고, 막히는 칸엔 ❓만 남겨두세요. 퍼실이 오면 그 지점부터 같이 풀어요.

```
내 도구가 없으면 누가 곤란한가요?        → ___________________________________
  └ 왜 곤란한가요? (1)                   → ___________________________________
     └ 지금 방식으론 왜 안 되나요? (2)     → ___________________________________
        └ 그래서 회사의 어떤 숫자가 나빠지나요? (3) → ______________________
```

예시 (우수 사례): 부스 담당자가 곤란해요 → 리드가 식기 전에 후속해야 하니까요 → 수작업으로는 48시간 안에 다 못 써요 → 그래서 **전환율**이 떨어져요.

세 번째 답에 **회사가 신경 쓰는 숫자**(전환율·처리시간·비용·이탈률)가 나왔나요? 안 나왔으면 그 칸에 ❓ 표시하고 넘어가세요.

---

## STEP 2 — 문제를 6칸으로 쪼개기

왼쪽에 내 답을 적고, 막히면 오른쪽 예시를 참고하세요. 그래도 막히면 ❓.

| # | 칸                     | 내 답 | 예시 (우수 사례)                                                                |
| - | ---------------------- | ----- | ------------------------------------------------------------------------------- |
| 1 | 누가 (구체적으로 누구) |       | 박람회 부스를 운영한 영업·마케팅 담당자                                        |
| 2 | 어떤 상황에서          |       | 박람회 직후 수십~수백 개 리드가 한꺼번에 쌓임                                   |
| 3 | 무슨 불편·고통        |       | 리드별 맞춤 후속 메일을 일일이 쓸 시간이 없음                                   |
| 4 | 얼마나 (크기·빈도)    |       | 48시간이 지나면 리드가 식어 전환율이 급락                                       |
| 5 | 기존 방법의 한계       |       | 템플릿 일괄발송은 개인화가 안 돼 열람·회신율이 낮음                            |
| 6 | 풀면 뭐가 달라지나     |       | 골든타임 안에 직책별 맞춤 메일 → 전환율↑ + 담당자는 우선순위 높은 리드에 집중 |

**5번이 제일 잘 비고, 그런데 제일 중요해요.** "기존 방식으로는 왜 안 되는지"를 못 말하면, 문제가 아니라 기능 자랑처럼 들리거든요. 여기서 막히면 꼭 ❓ 남겨두세요.

---

## STEP 3 — 결과물과 연결하기 (여긴 건너뛰지 마세요)

문제만 화려해지고 **내가 만든 것과 따로 놀면** 포폴에서 제일 먼저 티가 나요. 그래서 한 번 이어줘야 해요.

```
내가 만든 이 기능이(STEP 0), 방금 뾰족해진 이 문제의(STEP 2) 어느 지점을 정확히 푸나요?
→ ___________________________________________________________
```

이 한 문장이 잘 안 써진다면, 문제를 결과물이 감당하기 어려울 만큼 부풀린 거예요. STEP 2로 돌아가 결과물이 실제로 푸는 크기로 다시 맞춰보세요. (막히면 ❓)

---

## STEP 4 — 심사 4축 마무리, 딱 세 줄

기업이 실제로 눈여겨보는 부분이에요. 각 한 줄이면 충분해요. 막히면 ❓.

```
[판단] 나는 ______ 대신 ______ 을(를) 택했어요. 왜냐하면 ______.
   예시: 완전 자동발송 대신 '초안 생성'을 택했어요. 담당자가 검토·발송을 판단하게 해서 오발송을 막으려고요.

[검증] 제대로 되는지 ______ (으)로 확인했어요.
   예시: 실제 상담 메모 샘플을 넣어보며 직책별 톤이 맞는지 눈으로 확인했어요.

[주도성] 요구받은 것(______)을 넘어, ______ 까지 스스로 챙겼어요.
   예시: '메일 생성'만 요구됐지만, 직책별 개인화와 48h 골든타임 관점을 제가 더했어요.
```

---

## STEP 5 — 포트폴리오에 옮겨 담을 최종 문장 (3~4문장)

STEP 2·3·4를 이어 붙이면 그대로 완성돼요. 이 문장을 포폴 "문제 정의"란에 넣으면 됩니다.

```
[누가/상황] ____________________________________________은(는)
[문제+크기] ____________________________________________ 때문에,
[기존 한계] 기존 ______________________ 방식으로는 ______________________.
[해결+가치] 그래서 저는 ______________________ 을(를) 만들었고,
           그 결과 ______________________ (숫자 변화) + 담당자는 ______________________ (더 중요한 일) 하게 되었어요.
```

예시 (우수 사례): "박람회 부스 담당자는 직후 쌓인 수백 개 리드에 48시간 안에 맞춤 후속을 보내지 못해 전환 기회를 놓치고 있었어요. 기존 템플릿 일괄발송은 개인화가 안 돼 회신율이 낮았고요. 그래서 상담 메모로 직책별 맞춤 메일 초안을 만드는 파이프라인을 만들었고, 골든타임 안에 개인화된 후속을 보내 전환율을 높이면서 담당자는 우선순위 높은 리드 응대에 집중할 수 있게 되었어요."

---

### 다 채우면 이렇게 확인해요

퍼실이 이 문장을 읽고 **"아, 이건 진짜 문제네"** 라고 고개를 끄덕이고, **5·6번 칸과 STEP 4가 비어있지 않으면** 완성이에요. 퍼실에게 확인받고 ✔ 표시해요.

여기까지 오느라 고생했어요. 이 한 장이 곧 여러분 포폴의 첫 문단이 됩니다.
