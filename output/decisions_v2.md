## Case 01

**유형:** Slack DM

**기능명:** 최근 본 상품 목록 표시

**기능 목적:** 유저가 이전에 조회한 상품을 빠르게 재접근할 수 있도록 함

### 확정 요구사항
- 상품 상세 페이지에 최근 본 상품 목록 표시
- 비로그인 유저: 브라우저(로컬)에 저장
- 로그인 유저: 서버에 저장

### 의견/제안
- 최대 표시 개수: "10개? 20개? 잘 모르겠어요 일단 적당히" → 미결, 확정 아님
- 목록 삭제 기능: "있으면 좋죠 근데 급하진 않아요" → 확정 아님

### 정책
- **저장 정책:** 비로그인 → 브라우저 저장, 로그인 → 서버 저장

### 수치
- 최대 표시 개수: 미결

### 예외 상황
- 비로그인 상태에서 로그인 전환 시 브라우저 저장 목록 처리 방식 — 원문 언급 없음

**P2** — 긴급/필수 표현 없음

## Case 02

**유형:** CS 이메일

**기능명:** 구독 플랜 자체 변경

**기능 목적:** 유저가 직접 플랜을 변경할 수 있도록 하여 운영팀 부담 감소

### 확정 요구사항
- 유저가 직접 플랜 업그레이드 가능
- 유저가 직접 플랜 다운그레이드 가능
- 업그레이드: 즉시 적용
- 다운그레이드: 현재 구독 기간 만료 후 적용
- 플랜 변경 이력을 유저가 확인 가능

### 의견/제안
- 변경 이력 확인: "있으면 좋겠습니다" → 확정 표현 아니나 명확한 기능 요청으로 판단

### 정책
- **구독 정책:** 업그레이드 즉시 적용, 다운그레이드 기간 만료 후 적용
- **결제 정책:** 차액 정산 방식 — 미결

### 수치
- 없음

### 예외 상황
- 차액 정산 방식 미결 시 업그레이드 결제 처리 불가

### Priority
