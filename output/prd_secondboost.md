## [Case 01] 로그인 후 대시보드 자동 진입

### 기능 정의
로그인한 유저는 즉시 대시보드로 이동하며, 미인증 유저가 대시보드 URL에 직접 접근하면 로그인 페이지로 리다이렉트된다.

---

### 기능 목적
로그인 완료 후 유저가 핵심 화면인 대시보드를 즉시 확인할 수 있도록 하고, 인증되지 않은 접근을 차단한다.

---

### 사용자 시나리오
1. 유저가 로그인 화면에서 이메일/비밀번호를 입력하고 로그인한다.
2. 로그인 성공 시 대시보드 화면이 즉시 표시된다.
3. 대시보드에서 월별 매출 그래프를 확인한다.

**예외 시나리오**
1. 미인증 유저가 대시보드 URL(`/dashboard`)에 직접 접근한다.
2. 시스템이 인증 상태를 확인한다.
3. 로그인 페이지로 리다이렉트된다.

---

### 기능 요구사항

- **REQ-001** 로그인 성공 시 대시보드 화면을 즉시 표시한다.
- **REQ-002** 대시보드에 매출 그래프를 포함한다.
- **REQ-003** 매출 그래프는 월별 단위로 표시한다.
- **REQ-004** 미인증 유저가 대시보드 URL에 직접 접근 시 로그인 페이지로 리다이렉트한다.

---

### 정책

- **로그인 정책:** 미인증 상태에서 대시보드 접근 불가, 로그인 페이지로 강제 리다이렉트
- **권한 정책:** 로그인 유저만 대시보드 접근 허용

---

### 입력값

- 로그인 성공 이벤트 (인증 토큰)
- 미인증 유저의 대시보드 URL 직접 접근

---

### 출력값

- 대시보드 화면 (월별 매출 그래프 포함)
- 미인증 접근 시: 로그인 페이지로 리다이렉트

---

## [Case 02] 새 주문 푸시 알림 및 알림 설정

### 기능 정의
새 주문 발생 시 FCM 기반 푸시 알림을 전송하며, 유저는 알림 설정 화면에서 전체 on/off를 제어할 수 있다.

---

### 기능 목적
새 주문 발생을 유저에게 즉시 전달하고, 투자자 데모를 위한 핵심 기능으로 이번 스프린트 내 구현이 필요하다.

---

### 사용자 시나리오
1. 새 주문이 시스템에 접수된다.
2. 시스템이 FCM을 통해 해당 유저에게 푸시 알림을 전송한다.
3. 유저가 알림을 수신한다.

**알림 설정 시나리오**
1. 유저가 알림 설정 화면에 진입한다.
2. 전체 알림 on/off 토글을 조작한다.
3. 설정이 저장되고 이후 알림 발송에 반영된다.

---

### 기능 요구사항

- **REQ-001** FCM 기반 푸시 알림을 구현한다.
- **REQ-002** 알림 종류는 새 주문 1종으로 한정한다.
- **REQ-003** 알림 설정 화면을 제공한다.
- **REQ-004** 알림 설정은 전체 on/off만 지원한다.

---

### 정책

- **알림 정책:** 전체 on/off 설정 지원
- **알림 정책:** 알림 off 상태에서 중요 주문 알림 처리 방식 — 미결

---

### 입력값

- 새 주문 발생 이벤트
- 유저의 알림 설정 (on/off)

---

### 출력값

- FCM 푸시 알림 (알림 on 상태 유저 대상)
- 알림 설정 저장 결과

## [Case 03] 장바구니 상품 자동 만료 처리

### 기능 정의
장바구니에 담은 날로부터 7일 경과 시 해당 상품을 자동 삭제하며, 품절 상품은 즉시 삭제한다.

---

### 기능 목적
오래된 장바구니 상품을 정리하여 데이터 정합성을 유지하고, 품절 상품으로 인한 유저 혼선을 방지한다.

---

### 사용자 시나리오
1. 유저가 상품을 장바구니에 담는다.
2. 7일이 경과하면 시스템이 해당 상품을 장바구니에서 자동 삭제한다.
3. 유저가 장바구니를 확인하면 만료된 상품이 없다.

**품절 시나리오**
1. 장바구니에 담긴 상품이 품절 처리된다.
2. 시스템이 즉시 해당 상품을 장바구니에서 제거한다.

---

### 기능 요구사항

- **REQ-001** 장바구니에 담은 날로부터 7일 경과 시 해당 상품을 자동 삭제한다.
- **REQ-002** 품절 상품은 보관 기간(7일) 무관하게 즉시 장바구니에서 제거한다.

---

### 정책

- **저장 정책:** 장바구니 보관 기간 최대 7일 (담은 날 기준)
- **재고 정책:** 품절 상품은 보관 기간 무관 즉시 제거

---

### 입력값

- 장바구니 담기 이벤트 (담은 시각 기록)
- 상품 품절 이벤트

---

### 출력값

- 만료 상품 자동 삭제 처리
- 품절 상품 즉시 제거 처리

## [Case 05] 팀원 이메일 초대

### 기능 정의
이메일로 팀원을 초대하고, 미가입자는 회원가입 후 해당 팀에 자동 합류하며, 플랜별 인원 제한 및 7일 초대장 유효기간을 적용한다.

---

### 기능 목적
B2B 툴에서 팀 단위 협업을 지원하기 위해 팀원 초대 기능을 제공한다.

---

### 사용자 시나리오
1. 팀 관리자가 초대할 팀원의 이메일 주소를 입력하고 초대장을 발송한다.
2. 초대받은 유저가 이메일의 초대 링크를 클릭한다.
3. 미가입 유저는 회원가입 화면으로 이동하고, 가입 완료 후 해당 팀에 자동 합류한다.
4. 기가입 유저는 팀 합류 확인 후 바로 합류한다.

---

### 기능 요구사항

- **REQ-001** 이메일로 팀원 초대장을 발송한다.
- **REQ-002** 초대받은 미가입 유저는 회원가입 완료 후 해당 팀에 자동 합류한다.
- **REQ-003** 무료 플랜은 팀 최대 인원 5명으로 제한한다.
- **REQ-004** 유료 플랜은 팀 인원 제한이 없다.
- **REQ-005** 초대장 유효기간은 7일이며, 만료 후 초대 링크는 무효 처리한다.

---

### 정책

- **권한 정책:** 무료 플랜 팀 최대 5명, 유료 플랜 무제한
- **보안 정책:** 초대장 유효기간 7일, 만료 후 무효 처리

---

---

## [Case 15] 게시글 자동 저장

### 기능 정의
게시글 작성 중 30초마다 자동 저장하여 실수로 창을 닫아도 내용을 복구할 수 있도록 한다. 저장 위치(서버 vs 로컬)는 Conflict 상태로 Product Owner의 결정이 필요하다.

---

### 기능 목적
작성 중 창 닫힘, 새로고침 등의 사고로 인한 데이터 손실을 방지한다.

---

### 사용자 시나리오
1. 유저가 게시글을 작성한다.
2. 30초마다 자동 저장이 실행된다.
3. 유저가 실수로 창을 닫는다.
4. 다시 접속하면 작성 중이던 내용이 복구된다.

- **REQ-001** 게시글 작성 중 30초마다 자동 저장을 실행한다.

- **REQ-003** 저장 위치: Conflict 상태 — Product Owner 결정 후 확정 (아래 Conflict 참조)
